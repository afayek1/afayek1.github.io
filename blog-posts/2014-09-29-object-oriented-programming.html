Object Oriented Programming

September 29, 2014

Object oriented programming (OOP) is a paradigm that allows us to model the real world. Using OOP, we create objects which contain state and behavior contained in a single unit. This is in contrast to more traditional paradigms, such as the procedural paradigm, where programs are written step by step, and for the most part, go from top to bottom. This will make more sense as we go on, I promise…

Classes

Classes provide the blueprint for our object. A class is just a way of organizing and producing objects with similar attributes and methods. Take a look at our Stereo example below. Don’t try and understand all the code below, but take a look at it briefly and see if it makes any sense. Every time we create, or instantiate, an object from this class, they follow this basic blue print.



        class Stereo

          def initialize
            @current_volume = 0
            @source = "radio"
          end

          def change_source
            @source = "cd" if cd_button
            @source = "tuner" if tuner_button
          end

          def change_volume
            @current_volume-=1 if volume_down_button
            @current_volume+=1 if volume_up_button
          end

        end

Instance Variables

Objects instantiated from a class have have their own, individual copies of the variables specified in the class, meaning each instance of a class has it’s own, separate copy of it’s instance variables. For example, all Stereos may have an instance variable, volume, but each stereo’s volume can be changed without affecting another stereo’s volume. In Ruby, instance variables are denoted by @, as in @current_volume. Unlike a local variable which only exists within the method it is defined in (a concept known as scope), an instance variable can be accessed from anywhere within the object. Generally, when we want to store values, for example, to be accessed later, we store it in an instance variable which becomes part of the object’s state.
