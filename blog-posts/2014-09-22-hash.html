Hashes

September 22, 2014

A hash consists of unique keys which map to their value. For example, if we were looking at a menu, we may have something like this:

			 appetizers = {
				“egg rolls": 3.99,
				“chicken wings” =>5.99,
				“sliders” =>5.99,
				“mini tacos” => 4.99
			}

Notice that while our keys, ie. “egg rolls”, “chicken wings,” etc… are unique, the prices are not. So why is this useful? And how does it work? The topic itself is far more complex than I will get into, but this blog will serve as an introduction.


When keys are mapped to values, the key undergoes a hashing algorithm that essentially provides an address we can use to look up a value. These values are stored in buckets, and in an ideal situation, each key maps to a single address. Unfortunately, this is something we normally can’t avoid, but for simplicity we will assume these “collisions” don’t occur as there are ways to deal with them. Basically, the point we want to get across, is that given a key, we can immediately find it’s value by putting the key through a hashing algorithm. What this does is allow us to look up a value, given a key, instantly, and this is one of the main reasons we use hash maps.
