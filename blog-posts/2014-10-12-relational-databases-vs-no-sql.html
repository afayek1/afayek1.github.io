Difference between Relational and NoSQL Databases

October 12, 2014

Introduction

Database management Systems are based on database models. Database models determine the logical structure of the database, ie. how data can be stored, organizes, and manipulated. We will discuss two of the most popular database management systems, Relational Database Management Systems (RDBMS), often called SQL databases, and NoSQL databases.

Relational Databases

The RDBMS is based on the relational model. Relations have attributes and store data in tuples. Most relational databases use SQL to manage data, and are sometimes called SQL databases.

NoSQL Databases

NoSQL databases use one of multiple models, and can essentially be classified as “non-relational” databases, thus the "No" in NoSQL. The main models which represent how data is structured, are column, document, key-value, and graph. There are many NoSQL databases, each with a specified language used to manage its data.

Scalability

In a relational databases, the entire database has to be hosted on a single server. If relational databases want to handle more data, they must scale vertically, meaning they scale by adding more power to the existing machine (ex. adding more RAM). As a result, scaling is often limited to the capacity of a single machine, and scaling this way can be expensive. NoSQL databases on the other hand can be partitioned onto multiple machines, allowing for horizontal scaling, meaning they scale by adding more machines. As the price of storage has gone down significantly over time, the popularity of NoSQL has gone up.
